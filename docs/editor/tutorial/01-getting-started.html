Interstellar Armada Editor tutorial 01

Getting started

Creating the simplest mission

In this step-by-step walkthrough, we will create a one versus one battle between the player and a computer controlled enemy ship.

During the course of this tutorial, you will learn how to

  • create a new mission
  • set up teams for the mission
  • add ships to the mission
  • set which ships are controlled by the AI and by the player
  • export the mission to file
  • load and play the mission from the game

This tutorial is made for the Interstellar Armada Editor version 0.5.4.

  1. Creating the mission

    Click the New button in the top left corner.

    In the popup that appears, select "missions" as the Item type, if it is not already selected.

    Select "none" as the Copy from value, as we are going to create a whole new mission from scratch.

    Enter the name of your mission into the Name field. This will determine the name of the file when the mission is exported.

    Click Create.

    At this point you have successfully created a new mission, albeit it is an empty one without any spaceships. In the center, you should see the preview of the mission, and on the right side all the properties that you can edit to set up the mission. This is what we are going to continue with.

  2. Setting up teams

    The most basic mission needs to have two spaceships that are in different teams, so that they are hostile to each other. Before adding the ships, let's set up the two teams.

    By default, the teams property of the mission is not set. Click on the property (either its current value (not set) or the set button that appears when you hover it) to set it.

    The teams list is now set, but so far it is empty. Click on the empty list button to edit it.

    In the little popup that appears, click on the + button to add a new team.

    Now we have one team added. This will be the player's team, so enter "Player's team" in the name field, and hit enter.

    Click on the + button again to add a second team for the enemy.

    The list editor popup automatically switches to editing the new team when you add one, but you can always select which team you would like to edit using the dropdown at the top of the popup (now showing 2: Team 2).

    Enter "Enemy team" as the name for this second team and hit enter.

    Now you have the teams set up. You can close the list editor by clicking on its button (now showing [Player's team, Enemy team] again.

  3. Adding the player

    Similar to teams, now we will add the two spaceships. Unlike teams, the spacecrafts property is a required one, so it is already set to an empty list. Click on the empty list button to edit the list.

    In the popup, click on the + button to add the first spaceship.

    You will now see a lot of properties show up for the first spacecraft as well as you will see it appear in the middle of the preview (unless you have moved the camera of the preview). Don't worry, we only need a few things set for now, and will cover the rest of the properties in later tutorials.

    This first spacecraft will be our player, so let's set its name to "Player" by entering it in place of Spacecraft 1 and hitting enter.

    Click on the team property to set it. It's automatically set to the first team in the mission, which is the "Player's team", so we are good to go!

    For the class property, select "falcon". You should see the default container changing into a Falcon in the preview as you do.

    Click the piloted property to set it. A checkbox will appear next to it. Tick it to make this the ship that the player pilots in the mission.

  4. Adding the enemy

    Now it's time to add our enemy! Click the + button at the top of the spacecraft editor popup to add a new ship.

    Name this ship "Enemy" and set its team to the "Enemy team".

    For its class, choose "wolf".

    We need to make this ship piloted by the AI. Click on the ai property to set it. By default, it will be set to "fighter", which is exactly what we want, since this is a fighter.

    This enemy is now directly on the top of our player, which is not really ideal. Click on position to set it, and then set the position to 0, 100, 0 to put the enemy 100 meters in front of the player. You can see in the preview where the enemy will be located. (you need to hit enter or click somewhere outside the number editor after changing the values for the preview to refresh)

    You might want the enemy to face the player, so click on rotations to turn it around. Click on the none button that appears, then on + to add a new rotation, then choose the Z axis and enter 180 degrees (then hit enter). You should see in the preview the enemy now facing the player.

    We are now done with setting up the enemy, so you can close the spacecraft editor popup by clicking its button (now showing Spacecraft [2])

  5. Exporting the mission

    We now have our basic mission fully set up, so it's time to export it!

    Click on the Export button in the top left.

    The Item type will be automatically set to "missions" and the Item to the mission we just created.

    Enter your name into the Author field. Later if you wish to upload a (more interesting) mission to the Mission Hub for other players to try, you will need to use the same name here as for the Mission Hub, and for privacy reasons I recommend that you do not use your real name.

    Click Export. Your browser should now save a file with your entered mission name.

  6. Playing the mission

    If you have successfully exported a mission file, you can play it directly in the game.

    From the main menu, go to Single player / My missions, and click on the Choose file button.

    Select the file that you have saved. The mission will be loaded and you will see its details.

    Click Launch mission as you would with any normal mission to play it!
