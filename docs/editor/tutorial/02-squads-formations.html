Interstellar Armada Editor tutorial 02

Squads and formations

Creating an enemy squad and wingmen for the player

In this step-by-step walkthrough, we will update the mission we created in the first tutorial, expanding both the enemy and the player ships to whole squads with formations set up.

During the course of this tutorial, you will learn how to

  • import missions you have exported
  • set up squads
  • set up formations for squads
  • make the player the part of a squad
  • add individual ships to squads

This tutorial is made for the Interstellar Armada Editor version 0.5.4. We assume you have completed the first tutorial chapter and we will continue to edit the mission created there.

  1. Importing the mission

    If you do not have the mission open anymore as you restarted the editor since you created it, first you need to import it to continue editing.

    Click the Import button in the top left corner.

    In the popup that appears, the "missions" Item type will already be selected, as that is the only item type the editor can currently import.

    Click the Choose File button in the popup and then select the JSON file you exported, or just drag it on top of the button from your file explorer.

    Click Import.

    At this point you should see the mission in the state you have exported it, with the player ship facing an enemy fighter.

  2. Setting up the enemy squad

    It is more interesting to face multiple enemy fighters attacking in a formation rather than a single one, so let's first make that single enemy into a squad.

    First, select the enemy fighter, either by clicking on the button next to the spacecrafts property (Spacecraft [2]) in the right panel or clicking the fighter itself in the preview in the center.

    We can define a full squad with a single spacecraft entry. To do this, set the count property of the enemy spacecraft to 3. (as usual, click on the property value, the part that currently says single ship)

    Note that the available properties of the spacecraft change when you change the count to a number larger than one.

    At this point, we actually have 3 identical enemy fighters in the mission, but they do not belong to a squad yet and are placed right on top of each other, as we haven't set a formation for them yet.

    Let's add the enemy fighters to a squad. To do this, set the squad property (click on none). Before typing in a squad name, you can press the downward arrow or just click the text input again to bring up a list of suggestions. These suggestions are the built-in squad names that are automatically translated into the supported languages, so it is recommended to use them. Select (or type) "raider", a typical pirate squad name.

    At this point, the 3 enemy fighters belong to the "raider" squad, so later on they can be referenced together (such as when giving them commands), and they automatically get the names Raider 1, Raider 2, Raider 3.

  3. Setting the enemy formation

    Now let's set a formation for the raiders so they don't occupy the same point in space.

    Set the formation property of raider squad by clicking it, then bring up the options by clicking on the button that appears (wedge (0,0,0)).

    In the popup for editing the formation, you can choose a type and set a spacing. For now, leave the type as "wedge" and set a spacing of 50, -20, 0! You should see the enemy fighters spread out as you change the coordinates.

    Hint: you can experiment with different formation types by setting a higher ship count and a spacing along all 3 axes, then changing the formation type and checking how it looks in the preview. Don't set a too tight formation, as the ships will bump into each other as they try to maneuver!

    Note that ships of a squad defined this way in a single entry share some properties, such as class, AI type or position (aside from position within the formation), but there are several properties that you can still set as different for the squad members.

    At this point, you pretty much have the enemy squad set up! Time to add some friends!

  4. Setting up a friendly squad

    To even the odds against the 3 enemy fighters, now let's make a squad for the player as well!

    Select the player ship by clicking on it or selecting it in the spacecrafts list.

    Just like for the enemies, set its count to 3.

    Notice that as you have set the count, the enemies turned to green, as the player's spacecraft lost the "piloted" setting, since only a single ship can be set to be piloted, a group cannot!

    To fix this, set the newly appeared pilotedIndex property, which will tell which ship in the squad is piloted by the player. Set it to 1, giving the first spacecraft in this entry to the player.

    Just like for the enemies, set the squad property. Choose "alpha" this time, or come up with your own squad name and type it in, just make sure it does not have any spaces in it.

    Just like for the enemies, set the formation property to a "wedge" formation of 50,0,10 spacing, or to a formation of your choice!

    Lastly, we will need to set an AI for the player squad, since we want the wingmen of the player to be active. Don't worry, the fighter that is piloted by the player will not be taken over by the AI in this case. Set ai to "fighter".

  5. Adding an extra enemy to the enemy squad

    Let's say you want to add a spacecraft to a squad, but it is in a different position, has a different class or different AI type from the rest. You cannot do it from the same spacecraft entry, but you absolutely can by adding a new one.

    Let's add a fourth fighter to the enemy squad that is in a further away position.

    Add a new spacecraft, set its team to the enemy, its class to "wolf", ai to "fighter", its position to 0,200,0 and rotations to Z+180 degrees!

    Now this new enemy spacecraft is in a position that doesn't fit the formation of the raider squad, but you can still make it a part of it.

    Unset the name property (click the x button), then set the squad property to "raider 4". This will automatically make the fighter join the rest of the raider squad as the fourth spacecraft.

  6. Exporting the mission

    This concludes the second tutorial! At this point, you have a more interesting mission with multiple enemies and wingmen.

    Export and load the mission as explained in the first tutorial.

    Try adding multiple squads and maybe individual ships as well, experiment with different formations!
